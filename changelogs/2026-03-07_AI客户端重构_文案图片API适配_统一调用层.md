# AI客户端统一重构 — 文案API + 图片API适配

> **日期**: 2026-03-07
> **范围**: 文案生成和图片处理的AI API调用全面重构

---

## 背景

原代码中AI调用存在两个核心问题:
1. **图片生成API调用方式错误**: 使用 `chat.completions.create()` 发送图片URL，期望AI在文本中返回base64 — 这对 `gpt-image-1` 等图片模型不适用
2. **代码重复**: `stage1_pipeline.py` 和 `web/app.py` 各自创建独立的 OpenAI 客户端，切换供应商需改多处

## 解决方案

参考代理文档(yunwu.apifox.cn)，确认API规范:
- **文案**: `/v1/chat/completions` — 标准OpenAI chat格式
- **图片编辑**: `/v1/images/edits` — multipart/form-data，接受原图+prompt
- **图片创建**: `/v1/images/generations` — JSON body
- **响应格式**: 代理统一返回chat completion格式，base64在 `choices[0].message.content`
- **base_url**: `https://api.kk666.online/v1`

---

## 修改详情

### 1. 新建 `core/ai_client.py` — 统一AI客户端

| 函数 | 用途 | API端点 |
|------|------|---------|
| `get_openai_client()` | 获取OpenAI SDK客户端 | — |
| `ai_text(prompt, ...)` | 文案生成 | `/v1/chat/completions` |
| `ai_image_edit(image_data, prompt, ...)` | 图片编辑(传二进制) | `/v1/images/edits` |
| `ai_image_edit_url(image_url, prompt, ...)` | 图片编辑(传URL，自动下载) | `/v1/images/edits` |
| `ai_image_generate(prompt, ...)` | 纯文本生图 | `/v1/images/generations` |

- 图片API使用 `httpx` 直接调用（因代理返回非标准格式，OpenAI SDK无法解析）
- 响应解析兼容两种格式: 代理格式(`choices[0].message.content`) 和标准格式(`data[0].b64_json`)
- base64提取支持: data URI、Markdown图片、纯base64字符串

### 2. 更新 `stage1_pipeline.py`

- 删除 `self.client = OpenAI(...)` 实例创建
- `_ai_text()` → 委托给 `ai_text()`
- `_process_images()` → 改用 `ai_image_edit_url()` (下载图片→调用/v1/images/edits)
- 删除 `_ai_image_edit()` 和 `_ai_image_edit_url()` 两个旧方法（约80行）
- `_detect_product_type()` → 改用 `ai_text()`

### 3. 更新 `web/app.py`

- `/api/image-process` → 改用 `ai_image_edit_url()`，不再创建内联OpenAI客户端
- `/api/generate-copy` → 改用 `ai_text()`，不再创建内联OpenAI客户端
