## 修复 #4: ok() 空值判断 + 线程安全（P1）

**文件**: `web/app.py`

### 4a. ok() 函数空值判断

**问题**: `if data:` 当 `data={}` 或 `data=[]` 时不更新resp。

**修改**: `if data:` → `if data is not None:`

### 4b. Excel缓存线程安全

**问题**: `_excel_cache` 是普通dict，Flask多线程环境下有竞态条件。

**修改**:
- 新增 `_excel_cache_lock = threading.Lock()`
- 所有读写 `_excel_cache` 的地方用 `with _excel_cache_lock:` 包裹

**问题**: `task_status['result_file'] = result` 绕过了 `update_status()` 函数。

**修改**: 改为 `update_status(result_file=result)`

## 修复 #5: 配置热更新（P1）

**文件**: `config.py` + `web/app.py`

**问题**: Web页面修改API Key后保存到 `.env`，但运行中进程仍使用旧配置，提示"重启生效"。

**修改**:

### config.py — 重构为支持热更新的单例模式

```python
_config_instance = None

class Config:
    def __init__(self):
        self.reload()

    def reload(self):
        """重新加载环境变量"""
        load_dotenv(override=True)
        self.AI_API_KEY = os.getenv('AI_API_KEY', '')
        # ... 所有配置项 ...

def get_config():
    global _config_instance
    if _config_instance is None:
        _config_instance = Config()
    return _config_instance

def reload_config():
    """强制重新加载配置"""
    global _config_instance
    if _config_instance:
        _config_instance.reload()
    return _config_instance
```

### web/app.py — 保存配置后即时生效

- 导入 `reload_config`
- `/api/config` POST — 保存后调用 `reload_config()`，消息改为"已即时生效"
- `/api/sp-config` POST — 保存后调用 `reload_config()`，消息改为"已即时生效"
