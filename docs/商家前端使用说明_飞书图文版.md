# 亚马逊 2.8 商家前端使用说明（图文版）

> 适用范围：美国站商家前端 Web 使用
> 核心目标：让商家只通过页面就能完成“导入 -> 修字段 -> 预览 -> 提交 -> 看结果闭环”

## 0. 这份文档解决什么问题

这份文档只解决一件事：

**让商家只看前端页面，就知道该怎么把商品从 Excel 导入到 Amazon 提交完成。**

系统的核心主线很简单：

**导入数据 -> 修字段 -> 做诊断 -> 先预览 -> 再提交 -> 看结果闭环**

如果你第一次用，不要一上来就全量提交。建议先拿 1 个单品 SKU 和 1 个变体家族做演练。

---

## 1. 5 分钟快速上手

第一次上手，按这个顺序做就行：

1. 先去“设置”页配置美国站账号，并测试连接。
2. 回到“工作台”导入 Excel。
3. 看标题、主图、价格有没有缺失。
4. 需要的话先做 AI 文案 / AI 生图。
5. 先做“缺项诊断”。
6. 再做 Amazon 预览。
7. 预览通过后再正式提交。
8. 提交后去看表格状态、最近一次提交结果和任务中心。

如果你不确定应该走哪条流程，记这个判断：

- 已经有比较完整的商品 Excel：优先走“直接提交流程”
- 类目不确定、字段结构不稳定：优先走“官方模板流程”

## 2. 先认识页面：工作台是主战场

- 商品表格：SKU、标题、主图、价格、状态
- 右侧抽屉：查看和编辑单个商品详情

## 3. 第一次使用前：先去设置页把账号配好

点击左侧边栏的”设置”图标，进入设置页。这个页面分成两大块：**AI 配置**和**亚马逊账号**。

### 3.1 配置亚马逊账号（必做）

你需要准备 4 个凭证。下面是每个凭证在哪里获取：

| 凭证名称 | 在哪里找 | 长什么样 |
|---------|---------|---------|
| **Seller ID**（卖家 ID） | 登录 Seller Central → 右上角”设置” → “账户信息” → “商家令牌” | 格式如 `A2XXXXXXXXXX`，以 A 开头的一串大写字母数字 |
| **LWA Client ID**（应用 ID） | 登录 Developer Central → “应用信息” → “LWA 凭证” | 格式如 `amzn1.application-oa2-client.xxxxxx` |
| **LWA Client Secret**（应用密钥） | 同上，在 Client ID 旁边 | 一串随机字符串 |
| **Refresh Token**（刷新令牌） | 通过 SP-API OAuth 授权流程获取，通常由开发者生成后提供给你 | 格式如 `Atzr|xxxxxxx`，以 `Atzr|` 开头 |

> 如果你不是技术人员，让你的开发同事或 SP-API 对接方把这 4 个值给你就行。

**操作步骤：**

1. 在”亚马逊账号”区域点击**”添加”**按钮
2. 填写账号名称（自己起个名字方便识别，比如”美国站主账号”）
3. 选择站点（默认美国站 `Amazon US`）
4. 依次填入上面 4 个凭证
5. 点击**”保存”**
6. 在账号列表里找到刚添加的账号，点**”测试”**按钮
   - 显示绿色”连接成功” → 说明凭证正确，可以继续
   - 显示红色错误 → 检查凭证是否填错，尤其注意复制时有没有多余空格
7. 测试通过后，点**”设为默认”**

### 3.2 配置 AI（选做，如需 AI 文案/生图才配）

设置页上方的”AI 配置”区域，按下面步骤填写：

**第 1 步：获取 API Key**

1. 用浏览器打开 **api.kk666.online**
2. 注册/登录后，在后台找到你的 API Key（一串以 `sk-` 开头的密钥）
3. 复制这个 Key

**第 2 步：在设置页填写**

| 配置项 | 填什么 |
|--------|--------|
| **API Base（接口地址）** | `https://api.kk666.online/v1` |
| **API Key（密钥）** | 粘贴你在 api.kk666.online 获取的 Key |
| **文案模型** | 推荐填 `gpt-4o`（用来生成标题、五点、描述、搜索词） |
| **图片模型** | 推荐填 `gpt-4o`（用来给主图/副图换白底背景） |

**第 3 步：保存并测试**

1. 填好后点**”保存”**
2. 再点**”测试连接”**
3. 看到绿色”连接成功”就说明配好了

> 如果你只做提交不做 AI 生成，可以跳过这一步，直接配亚马逊账号即可。

### 3.3 最重要的一条

**没有配好亚马逊账号并测试通过前，不要直接开始提交。**

## 4. 直接提交流程：商家最常用的主流程

这条流程适合**已经有比较完整商品 Excel** 的商家。从导入到提交，一步步来。

### 4.1 第一步：导入 Excel

**操作方法：**

1. 在工作台页面，点击工具栏左侧的**"导入 Excel"**蓝色按钮
2. 选择你的商品文件（支持 `.xlsx` 和 `.xls` 格式）
3. 等待上传完成，页面会自动刷新出商品列表

**上传成功后，先别急着操作，花 1 分钟检查这 5 件事：**

| 检查项 | 怎么看 | 不正常怎么办 |
|--------|--------|-------------|
| SKU 是否识别 | 表格第一列应该显示每个商品的 SKU | 检查 Excel 里是否有"SKU"列 |
| 标题是否显示 | 表格第二列应该有商品标题 | 检查 Excel 里是否有"标题"或"item_name"列 |
| 主图是否能看到 | 表格第三列应该有缩略图 | 确认图片 URL 是公网可访问的链接 |
| 价格是否正常 | 表格应该有价格列 | 检查 Excel 里是否有"价格"或"standard_price"列 |
| 变体分组是否正确 | 如果有父子商品，应该自动分组显示 | 检查 `parent_sku` 和 `parentage_level` 列 |

如果这一步就发现明显异常（比如所有标题都是空的），说明 Excel 列名和系统不匹配，先修 Excel 再重新导入。

### 4.2 第二步：熟悉表格操作

表格上方有一排工具：

- **搜索框** — 输入 SKU 或商品名快速定位
- **筛选按钮** — 点"全部"/"待处理"/"已处理"/"已提交"等快速过滤
- **分页** — 每页默认显示 50 条，可以在右下角切换
- **列设置** — 点齿轮图标，勾选想看的列（比如打开"预览状态""ASIN"列）

**推荐先做一件事：用筛选找出有问题的行**

1. 点击**"校验错误"**筛选按钮 → 看有没有标题、价格、主图缺失的行
2. 如果有，先修好再继续后面的步骤

### 4.3 第三步：用抽屉编辑单个商品

**打开方式：** 在表格里点击任意一行右边的**"编辑"**按钮，右侧会滑出一个抽屉面板。

**抽屉里有这些 Tab：**

| Tab 名称 | 干什么用 |
|----------|---------|
| **文案** | 看原始标题和 AI 标题的对比，编辑标题、五点、描述、搜索词 |
| **图片** | 看原始主图和 AI 图的对比，点"生成"重新换背景，点"快速预览"试看效果 |
| **品类字段** | 编辑颜色、尺寸、材质等属性字段 |
| **诊断** | 看缺项诊断结果（缺哪些字段、Amazon 预览有什么问题） |

**修改字段的流程：**

1. 在抽屉里找到要修改的字段
2. 直接在输入框里修改
3. 修改完后点底部的**"保存"**按钮
4. 看到绿色提示"已保存到文件"说明成功
5. 回到表格，数据会自动刷新

**建议：先拿 1 条 SKU 在抽屉里把编辑 → 保存 → 刷新跑通，确认没问题后再做批量。**

## 5. AI 文案 / AI 生图：什么时候该用，怎么操作

> 前提：需要先在设置页配好 AI（见第 3.2 节）。如果你的商品数据已经够好，可以跳过 AI 直接做诊断。

### 5.1 单条 AI 文案（推荐先试 1 条）

**操作步骤：**

1. 在表格里点击某个 SKU 的**"编辑"**按钮，打开抽屉
2. 切到**"文案"**Tab
3. 你会看到每个字段（标题、卖点 1-5、描述、搜索词）都有**"原始"**和**"AI 生成"**两栏
4. 点击某个字段旁边的**"重新生成"**按钮
5. 等待几秒，AI 结果会出现在右侧"AI 生成"栏
6. 觉得满意就点**"采用"**按钮，AI 内容会替换到正式字段
7. 点底部**"保存"**写入文件

### 5.2 批量 AI 文案

**操作步骤：**

1. 在表格里勾选要处理的 SKU（点每行左边的复选框）
2. 点工具栏的**"批量 AI 文案"**按钮
3. 选择生成范围（全部字段 / 只标题 / 只五点 / 只描述 / 只搜索词）
4. 点**"开始"**
5. 页面会出现进度条，显示当前处理到第几个 SKU
6. 处理完后会弹出完成提示，任务中心也会有记录

> 如果中途想停，点进度条上的**"取消"**按钮。已处理的部分会保留，不会丢。

### 5.3 单条 AI 生图（换白底背景）

**操作步骤：**

1. 打开某个 SKU 的抽屉，切到**"图片"**Tab
2. 你会看到原始主图和 AI 主图的对比
3. 先选择背景风格：**纯白** / **生活场景** / **渐变**（一般选纯白）
4. 点**"生成"**按钮 → 系统会调用 AI 重新生成主图
5. 生成完成后，新图会出现在右侧
6. 觉得满意就点**"采用"**

### 5.4 批量 AI 生图

1. 勾选要处理的 SKU
2. 点工具栏的**"批量 AI 生图"**按钮
3. 选择范围（只主图 / 全部图片）和背景风格
4. 点"开始"，等待进度条走完

### 5.3 快速换背景预览

抽屉的图片 Tab 新增了"快速预览"按钮：

- 点击后会用当前原图直接调用 AI 换背景
- 弹窗展示 AI 生成结果，不走完整 pipeline
- 适合快速验证换背景效果后再决定是否正式生成

### 5.4 AI 合规自动校验（v0.3.0 新增）

AI 生成内容后，系统会自动做 Amazon 官方规范校验：

| 字段 | 自动校验项 |
|------|-----------|
| 标题 | 200 字符限制、禁用字符（`! $ ? _ { } ^` 等）、同词不超过 2 次 |
| 五点 | 500 字符限制、禁止价格/运费/退款信息、禁止 emoji |
| 搜索词 | 按站点字节限制（美国 250 / 日本 500）、空格不计入、自动去重标题和五点中已有的词 |
| 图片 | 最长边 ≥ 1000px、主图白底采样校验、自动转 JPEG |

如果检测到问题，标题会自动修正（删除禁用字符、去重复词），搜索词会自动截断。

### 5.5 AI 完成后一定要再检查 3 件事

- 标题是否仍然符合商品本身
- 主图是否清晰、可访问
- 图片是否已经变成可提交的公开地址

注意：

**本地图片路径不能直接提交给 Amazon。**

## 6. 缺项诊断：正式预览前一定要先跑

缺项诊断的作用不是“替代提交”，而是：

**提前把明显会失败的问题找出来。**

系统通常会帮助你发现：

- 缺少必填字段（比如标题、品牌、UPC）
- 图片不可访问（URL 打不开）
- 本地规则不通过（标题太长、搜索词超限等）
- 模板阻断项未修复
- 变体父子关系有问题

**v0.3.0 新增：诊断弹窗现在展示完整 5 层明细**

点击"缺项诊断"后，弹窗会按 SKU 分别展示：

1. **本地校验** — 标题禁用字符、搜索词超限、缺必填字段等
2. **Amazon 预览** — Amazon 返回的错误码、涉及字段、修复建议
3. **Listing 后台问题** — 如果该 SKU 已存在，显示当前后台 issues
4. **媒体连通性** — 每张图片 URL 是否可以正常访问
5. **统计概览** — 错误数、警告数、缺失字段数、媒体失败数

每个 issue 都会标注严重程度（红色 = 阻断，黄色 = 警告），方便你判断优先修哪个。

推荐顺序是：

1. 先做缺项诊断
2. 看明细，优先修红色阻断项
3. 然后做 Amazon 预览

这样返工最少。

## 7. Amazon 预览：正式提交前最关键的一步

Amazon 预览的意思是：把你的商品数据发给亚马逊，让亚马逊**检查一遍格式对不对**，但**不会真的上架**。这样你可以提前发现问题。

### 7.1 怎么做预览

**操作步骤：**

1. 在表格里勾选要预览的 SKU（第一次建议只选 1 个）
2. 点工具栏右侧的**"Amazon 预览"**按钮
3. 弹出确认框，检查：
   - **目标账号** — 是不是你要用的那个卖家账号
   - **目标站点** — 是不是美国站
   - **SKU 数量** — 是不是你想预览的数量
4. 确认无误后点**"开始预览"**
5. 等待系统逐个发送给亚马逊检查

### 7.2 怎么看预览结果

预览完成后会弹窗显示每个 SKU 的结果：

| 状态 | 含义 | 下一步 |
|------|------|--------|
| **VALID**（绿色） | 数据格式通过，可以正式提交 | 直接做正式提交 |
| **INVALID**（红色） | 有必须修复的错误 | 看错误详情，修好后重新预览 |
| **ERROR** | 接口调用失败 | 检查网络或账号凭证 |

**预览没通过的 SKU，绝对不要直接正式提交。** 先根据错误提示修字段，修好后重新预览。

## 8. 正式提交：先小批量，再放量

只有预览通过（`VALID`）的 SKU 才建议做正式提交。

### 8.1 怎么做正式提交

**操作步骤：**

1. 勾选要提交的 SKU（第一次建议 1~3 个）
2. 点工具栏的**"正式提交"**按钮
3. 弹出确认框，仔细检查：
   - **操作类型** — 确认显示的是"正式提交"而不是"预览"
   - **目标账号** — 和预览时用的是同一个
   - **SKU 数量** — 是你想提交的数量
4. 确认后点**"开始提交"**
5. 系统会逐个把商品提交到亚马逊

### 8.2 提交后看什么

提交完成后，不要关页面，先看这几个地方：

1. **弹窗结果** — 会列出每个 SKU 的提交状态
2. **表格里的"提交状态"列** — 显示 `ACCEPTED` 或错误原因
3. **表格里的"ASIN"列** — 如果亚马逊分配了 ASIN 会显示在这里

> `ACCEPTED` 只代表亚马逊**接受了你的请求**，不代表商品已经上架。需要继续在第 9 章看最终状态。

### 8.3 第一次提交的建议

| 做法 | 建议 |
|------|------|
| 第 1 次 | 只提交 1 个单品 SKU |
| 第 2 次 | 提交 1 个变体家族（父体 + 2-3 个子体） |
| 第 3 次 | 确认没问题后，扩大到 10-20 个 SKU |
| 之后 | 逐步放量到正常批量 |

**不建议第一次就整批提交几十个商品。**

## 10. 官方模板流程：什么时候该走这条路

如果你遇到下面情况，更适合用模板流程：

- 不确定该用哪个 Amazon 类目模板
- 字段缺得比较多
- 想先把结构标准化再填数

推荐顺序：

1. 用链接 / 标题 / 关键词识别类目
2. 生成官方模板
3. 下载模板并填数
4. 上传模板结果
5. 自动诊断
6. Amazon 预览
7. 正式提交

要点：

- 模板不是目的，模板只是把字段准备到可提交状态的工具
- 模板上传后如果还有阻断项，先修阻断，再预览，再提交

说明：

## 11. 变体家族流程：不要把父子关系当普通单品处理

变体商品一定要把家族结构理顺后再操作。

### 11.1 先记住 4 个核心字段

- `parentage_level`
- `parent_sku`
- `variation_theme`
- 颜色 / 尺码等变体维度字段

### 11.2 正确顺序

1. 先确认父体存在
2. 再确认每个 child 都挂对父体
3. 检查父体和子体的 `variation_theme` 是否一致
4. 检查颜色 / 尺码维度是否完整
5. 先预览整个家族

## 12. 商家最容易踩坑的地方

### 12.1 主图不是图片直链

错误示例：

- 本地路径
- 商品详情页链接
- 需要登录才能访问的图片地址

正确要求：

- 公网可访问
- 能直接打开图片内容

### 12.2 SKU 重复

如果同一个文件里有重复 SKU，会直接影响提交和结果判断。

建议每次提交前都先检查 SKU 唯一性。

### 12.3 标题、价格、主图缺失

这三项是最常见的阻断问题。

### 12.4 变体结构没修好就强行提交

这会导致：

- 预览不过
- 家族提交异常
- 结果判断混乱

### 12.5 以为 ACCEPTED 就等于完全成功

并不是。

`ACCEPTED` 只是说明 Amazon 已接受请求，后面还需要继续看 listing 的最终状态。

## 14. 常见问题 FAQ

### Q1：必须先用模板流程吗？

不必须。已有比较完整的 Excel，优先走直接提交流程。

### Q2：AI 是必须的吗？

不必须。如果原始标题、卖点、图片已经可用，可以直接做诊断、预览和提交。

### Q3：提交后没立刻拿到 ASIN，算失败吗？

不一定。有些情况需要继续看任务记录和最终 listing 状态。

### Q4：变体商品能不能只提一个 child？

不建议。优先按完整 family 处理。

### Q5：为什么正式提交前还要先预览？

因为预览可以提前暴露 Amazon 会拒绝的问题，减少正式提交后的返工。

## 16. 术语表：不懂的词看这里

| 术语 | 通俗解释 |
|------|---------|
| **SKU** | 你自己给商品起的编号，比如"BLK-SHIRT-XL"，每个商品必须有唯一的 SKU |
| **ASIN** | 亚马逊给商品分配的编号，提交成功后才会拿到，格式如 `B0XXXXXXXX` |
| **UPC / EAN / GTIN** | 商品条形码，买的时候就有，没有的话需要申请豁免 |
| **Seller ID** | 你在亚马逊的卖家账号 ID，在 Seller Central 账户信息里查看 |
| **LWA Client ID / Secret** | 用来让系统代替你调用亚马逊接口的"钥匙"，在开发者后台获取 |
| **Refresh Token** | 授权令牌，让系统能持续访问你的亚马逊账号，由开发者通过授权流程生成 |
| **SP-API** | Amazon Selling Partner API，亚马逊官方接口，系统通过它帮你提交商品 |
| **product_type** | 亚马逊的商品类目，比如 `WIRELESS_ACCESSORY`（无线配件），不同类目要求的字段不同 |
| **parentage_level** | 变体层级，`parent` = 父体（商品主体），`child` = 子体（具体规格，如红色 XL） |
| **variation_theme** | 变体主题，告诉亚马逊你的子体之间区别在哪，比如 `COLOR_NAME`（颜色区分） |
| **ACCEPTED** | 亚马逊已接受你的提交请求，但**不等于**已经上架成功，需要继续看最终状态 |
| **VALID** | 预览验证通过，说明数据格式没问题，可以正式提交 |
| **INVALID** | 预览验证不通过，有必须修复的错误 |
| **缺项诊断** | 正式预览前的自检，帮你提前发现缺字段、图片不通等问题 |
| **Bullet Points** | 亚马逊商品页面上的"五点描述"，每条用来突出一个产品卖点 |
| **Search Terms** | 后台搜索关键词，买家搜索时用来匹配你的商品，不会显示在商品页面上 |
| **media locator** | 图片的提交地址，通常是 `https://` 或 `s3://` 开头的链接 |

如果你之前已经在用这个系统，以下是最近更新的要点：

### 新增功能

## 18. 一句话总结

对商家来说，最重要的不是记住技术细节，而是记住这条主线：

**导入数据 -> 修字段 -> 做诊断 -> 先预览 -> 再提交 -> 看结果闭环。**

如果你不知道下一步该做什么，优先看：

- 有没有阻断项（红色标记）
- Amazon 预览是否通过（状态是不是 VALID）
- 最近一次任务记录是什么（看任务中心）
