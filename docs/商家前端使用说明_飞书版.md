# 亚马逊 2.8 商家前端使用说明（飞书版）

## 1. 这份说明适合谁

这份说明面向两类人：

- 直接使用系统上架商品的运营同学
- 需要先整理采集数据、再交给运营提交的助理同学

当前系统以 **美国站** 为主，默认使用 **Web 页面** 完成全部操作，不需要 CLI。

一句话理解这个工具：

**把“采集数据 -> AI 改写/改图 -> 缺项诊断 -> Amazon 预览 -> 正式提交”串成一条可操作的前端流程。**

---

## 2. 你在系统里会看到的两条主流程

### 流程 A：直接提交流程

适合场景：

- 你已经有比较完整的商品 Excel
- 你想直接导入后修字段、做预览、正式提交

推荐顺序：

1. 导入 Excel
2. 检查表格和抽屉字段
3. 需要的话执行 AI 文案 / AI 生图
4. 执行缺项诊断
5. 执行 Amazon 预览
6. 通过后正式提交
7. 查看提交结果和最终状态

### 流程 B：官方模板流程

适合场景：

- 你不确定商品该用哪个 Amazon 类目模板
- 你希望先生成标准模板，再把数据填进去

推荐顺序：

1. 用链接 / 标题 / 关键词识别类目
2. 生成官方模板
3. 下载模板并填数
4. 上传模板结果
5. 自动诊断
6. Amazon 预览
7. 正式提交

建议：

- **已有完整商品数据时，优先走“直接提交流程”**
- **字段不完整、类目不确定时，再走“官方模板流程”**

## 3. 页面结构说明

系统主要分为两个页面：

### 3.1 工作台

这是日常最常用的页面，核心区域包括：

- 顶部上下文栏：当前文件、默认账号、模板、product_type
- 统计栏：总商品、AI 已处理、待处理、校验错误、已提交、当前 ASIN、最终可发现
- 官方字段模板区域：识别类目、生成模板、上传模板结果
- 任务中心：查看当前任务和最近记录
- 操作工具栏：导入、下载模板、导出、批量 AI、校验、提交
- 商品表格：查看 SKU、标题、图片、价格、状态
- 右侧抽屉：查看和编辑单个商品详情

### 3.2 设置页

主要用于：

- 配置亚马逊账号
- 测试账号连接
- 选择默认账号
- 配置 AI 文案 / AI 图片参数
- 配置并发和输出参数

## 4. 第一次使用前，先做这 3 件事

### 4.1 配置亚马逊账号

进入“设置”页面后：

1. 添加账号
2. 填写 Seller ID、LWA Client ID、LWA Client Secret、Refresh Token
3. 选择美国站账号
4. 点击“测试连接”
5. 测试通过后设为默认账号

建议：

- 先只保留一个主账号，降低误操作概率
- 如果有多个账号，账号名称要写清楚用途，例如“美国站主账号”

### 4.2 检查 AI 配置

如果你要用 AI 文案或 AI 生图，需要在设置页确认：

- 文本模型配置已保存
- 图片模型配置已保存
- 测试接口可正常返回

### 4.3 准备商品文件

建议 Excel 至少包含这些关键字段：

- `SKU`
- `item_name` 或标题
- `main_image_url`
- `price` / `standard_price`
- `product_type`（如果已知）
- `brand`

如果是变体商品，还要尽量补齐：

- `parent_sku`
- `parentage_level`
- `variation_theme`
- 颜色 / 尺码等变体属性

## 5. 工作台主流程：直接提交流程

## 5.1 导入 Excel

在工作台点击“导入 Excel”：

1. 选择 `.xlsx` 或 `.xls`
2. 导入成功后，表格会显示商品列表
3. 顶部上下文会显示当前文件名

导入后第一时间检查：

- SKU 是否正常显示
- 标题是否识别正确
- 主图是否正常显示
- 价格是否识别成功
- 变体父子关系是否分组正确

### 5.2 浏览表格与筛选商品

你可以在表格里做这些操作：

- 搜索 SKU / 商品名
- 按状态筛选
- 调整列显示
- 勾选部分商品做批量操作
- 点击某一行打开右侧抽屉

建议先检查异常行：

- 标题为空
- 主图为空
- 价格为空
- 变体关系异常

### 5.3 在抽屉里修字段

点击商品行后会打开右侧抽屉。

常见操作：

- 修改标题、卖点、描述
- 修改价格、库存、状态
- 修改图片地址
- 查看缺项诊断结果
- 查看提交流程状态
- 对单个商品执行 AI 文案 / AI 生图 / 预览 / 提交

建议：

- 抽屉是修单个问题的最佳入口
- 批量前先用抽屉把一条样例 SKU 跑通

### 5.4 AI 文案 / AI 生图（可选）

如果采集来的内容需要优化：

- 批量 AI 文案：适合批量重写标题、卖点、描述
- 批量 AI 生图：适合统一重生主图或全部图片
- 单条 AI：适合先拿样例商品验证效果

建议操作方式：

1. 先选 1 个 SKU 测试
2. 确认结果满意后再做批量
3. AI 完成后重新检查主图、标题和卖点是否可用

注意：

- AI 图片要能生成可提交的图片地址
- 本地路径不能直接拿去提交 Amazon

### 5.5 缺项诊断

正式预览前，建议先跑一次“缺项诊断”。

系统会帮助你提前发现：

- 缺少必填字段
- 图片不可访问
- 本地规则不通过
- 模板阻断项
- 变体父子关系异常

这一步的目标不是直接提交，而是：

**把明显会失败的问题前移暴露出来。**

### 5.6 Amazon 预览

这是正式提交前最关键的一步。

建议：

- 先对单个 SKU 做预览
- 通过后再批量预览
- 预览失败时，先看错误提示，不要直接正式提交

预览通过后，你应该重点确认：

- 状态是否为 `VALID`
- 是否还有 ERROR 级别问题
- 是否仍有变体结构问题

### 5.7 正式提交

预览通过后，再做正式提交。

提交前弹窗会显示：

- 目标账号
- 目标站点
- 提交模式
- SKU 数量

提交成功后，系统会展示：

- 提交状态
- 提交 ID
- 提交路由
- ASIN（如已拿到）
- 最近一次提交结果
- 任务记录

建议：

- 第一次正式提交时，先用 1~3 个 SKU 小批量验证
- 不建议第一次就整批提交几十上百个 SKU

## 6. 官方模板流程

如果你不确定字段该怎么填，或者想先标准化结构，可以走模板流程。

### 6.1 识别类目

你可以输入：

- 商品链接
- 商品标题
- 搜索关键词

系统会尝试推荐适合的 `product_type`。

### 6.2 生成模板

识别后可以：

1. 选择单品模板或父子变体模板
2. 点击生成模板
3. 下载模板文件

模板适合用于：

- 给采集插件回填
- 给运营手工补字段
- 做标准化商品录入

### 6.3 上传模板结果

模板填好后回到工作台上传。

上传后系统会自动执行：

- 模板检查
- 缺项诊断
- Amazon 预览相关检查

### 6.4 修阻断项后再提交

模板流程的关键不是“下载模板”，而是：

**通过模板把字段准备到可提交状态。**

如果上传后仍有阻断项：

- 优先修模板阻断问题
- 再跑预览
- 最后做正式提交

## 7. 变体家族使用说明

变体商品请重点关注父体和子体结构。

### 7.1 基本概念

- `parent`：父体，不直接卖，只负责组织家族
- `child`：子体，真正提交和售卖的商品
- `parent_sku`：子体对应的父体 SKU
- `variation_theme`：变体主题，例如颜色、尺码、颜色+尺码

### 7.2 变体操作建议

正确顺序：

1. 先检查父体是否存在
2. 检查子体是否都挂到正确父体
3. 检查 variation_theme 是否一致
4. 检查颜色 / 尺码等维度是否完整
5. 先预览整个家族
6. 通过后再提交整个家族

### 7.3 家族按钮怎么用

在家族区域你会看到类似按钮：

- 预览整个家族
- 提交整个家族
- 编辑家族矩阵
- 修复问题定位

建议：

- 不要只提交单个 child，优先按完整 family 处理
- 发现父体缺失、theme 不一致、维度缺失时，先修结构再提交

## 8. 提交后怎么看结果

提交完成后，前端主要有 4 个地方可以看结果：

### 8.1 表格行内状态

重点看这些列：

- 提交状态
- 提交 ID
- 提交路由
- ASIN

### 8.2 行内摘要 / 抽屉状态

你可以看到：

- 最近一次提交状态
- 是否拿到 ASIN
- 路由是已有 ASIN 还是新建 listing

### 8.3 最近一次提交结果 Banner

页面顶部会显示最近一次提交的摘要，适合快速复查：

- 哪个文件
- 哪个账号
- 哪个站点
- 哪个 SKU
- 当前状态

### 8.4 任务中心

任务中心适合看：

- 当前正在执行什么
- 最近一次预览 / 提交结果
- 某次任务的详细记录

## 9. 常见状态怎么理解

### 9.1 预览状态

- `VALID`：预览通过，可以进入正式提交
- `INVALID`：预览未通过，先修问题
- `ERROR`：请求或处理异常，需要重试或检查配置

### 9.2 提交状态

- `ACCEPTED`：Amazon 已接受请求，进入后续处理
- `PREVIEW_BLOCKED`：前置检查未通过，未进入正式提交
- `ERROR`：请求失败或处理异常
- `SKIPPED`：本次未执行，例如缺 SKU、重复 SKU

### 9.3 最终状态

- `DISCOVERABLE`：商品已进入可发现状态
- 其他状态：说明 Amazon 后台还在处理，或仍有后续问题

注意：

**`ACCEPTED` 不等于最终一定上架成功。**

它表示 Amazon 已接受请求，还需要继续看最终 listing 状态。

## 10. 商家最容易踩坑的地方

### 10.1 主图不是可访问的图片直链

错误示例：

- 本地路径
- 网页详情页链接
- 需要登录才能打开的地址

正确要求：

- 公网可访问
- 能直接打开图片内容

### 10.2 SKU 重复

如果同一个文件里有重复 SKU，会影响提交和结果判断。

建议：

- 每个 SKU 保持唯一
- 正式提交前先检查重复值

### 10.3 价格 / 标题 / 主图缺失

这是最常见的阻断项。

建议在导入后第一时间先看：

- 标题
- 主图
- 价格

### 10.4 变体结构不完整

常见问题：

- 子体没有 parent_sku
- 父体不存在
- 父体和子体 variation_theme 不一致
- 颜色 / 尺码维度不完整

### 10.5 以为提交成功就已经上架完成

实际上：

- 先是预览通过
- 然后正式提交
- 然后 Amazon 后台继续处理
- 最终还要看 listing 状态

## 11. 推荐的日常使用 SOP

如果你是普通运营，建议固定用下面这套顺序：

### 单品 SOP

1. 导入文件
2. 抽查一条 SKU
3. 必要时做 AI 文案 / AI 图片
4. 跑缺项诊断
5. 先预览 1 条
6. 再预览全部
7. 正式提交小批量
8. 看任务中心和最终状态

### 模板 SOP

1. 识别类目
2. 生成模板
3. 填模板
4. 上传模板结果
5. 修阻断项
6. 做 Amazon 预览
7. 正式提交

### 变体 SOP

1. 检查父子结构
2. 修矩阵
3. 预览整个家族
4. 提交整个家族
5. 核对每个 child 的结果

## 12. 常见问题 FAQ

### Q1：我必须先用模板流程吗？

不必须。

如果你已有较完整的商品 Excel，优先走直接提交流程就可以。

### Q2：AI 是不是必须做？

不必须。

如果原始标题、卖点、图片已经可用，可以直接做诊断、预览和提交。

### Q3：提交后没立刻拿到 ASIN，算失败吗？

不一定。

有些情况需要再看任务记录和最终 listing 状态。

### Q4：变体商品能不能只提一个 child？

不建议。

建议优先按完整 family 处理，避免父子关系不完整导致问题。

### Q5：为什么我点正式提交前还要先预览？

因为预览能提前暴露 Amazon 会拒绝的问题，能减少正式提交后的返工。

## 13. 给运营主管的落地建议

如果你要让团队开始正式使用，建议这样安排：

1. 先选 1 个美国站账号做试运行
2. 先选 1 个单品 + 1 个变体家族做样例
3. 先做小批量，不要一上来大批量
4. 先固化 SOP，再放开给更多同事
5. 让每次提交都保留文件名、账号、提交记录，方便追溯

---

## 14. 一句话总结

对商家来说，最重要的不是记住所有技术细节，而是记住这条主线：

**导入数据 -> 修字段 -> 做诊断 -> 先预览 -> 再提交 -> 看结果闭环。**

如果你不确定下一步该做什么，优先看：

- 当前有没有阻断项
- Amazon 预览是否通过
- 任务中心最近一次记录是什么
