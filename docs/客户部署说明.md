# 亚马逊 2.8 客户交付与新电脑部署说明

这份说明面向交付/售后人员。客户不需要理解 Python、pip、依赖、虚拟环境，客户只拿到一个压缩包，解压后双击启动脚本即可使用。

## 推荐交付方式

1. 在 GitHub Release 下载客户电脑对应的压缩包：
   - macOS Apple Silicon：`AmazonListingTool-darwin-arm64.zip`
   - Windows 64 位：`AmazonListingTool-windows-amd64.zip`
2. 把 zip 发给客户，让客户先解压，再双击：
   - macOS：`启动亚马逊2.8.command`
   - Windows：`启动亚马逊2.8.bat`
3. 程序会自动：
   - 检测运行环境
   - 创建 `.env`、`accounts.json`、`input/`、`output/`、`logs/`
   - 启动本地 Web 服务
   - 打开浏览器访问工作台
4. 第一次使用只需要在「设置」页填写：
   - AI 文案 Key / 图片 Key
   - Amazon SP-API 账号信息

## 客户电脑不需要安装的东西

客户电脑不需要安装以下内容：

- Python
- pip
- `requirements.txt`
- 虚拟环境 `.venv`
- PyInstaller
- 任何命令行依赖

这些内容已经被打包进 `AmazonListingTool` 可执行程序里。

## 环境检测

发行包内置环境检测入口：

- macOS：双击 `环境检测.command`
- Windows：双击 `环境检测.bat`

检测项包括：

- 当前操作系统和架构
- 是否为已打包发行版
- `input/output/logs/config` 目录是否可写
- `.env` 是否存在
- `accounts.json` 是否存在、默认账号是否填完整
- AI Key 是否填完整
- Web 端口是否可用

检测结果只有 `ERROR` 才会阻止启动；`WARN` 表示可以打开软件，但某些业务功能可能需要配置后才能使用。

## 端口防呆

默认端口是 `5000`。如果客户电脑上已经有其他软件占用了 `5000`，发行版会自动尝试 `5001`、`5002` 等后续端口，并在控制台输出实际访问地址。

## 常见问题处理

### 1. 双击没反应或打不开

先双击 `环境检测.command` / `环境检测.bat`，让客户截图发给技术支持。

### 2. 浏览器没有自动打开

查看启动窗口里的地址，通常是：

```text
http://127.0.0.1:5000
```

如果端口被占用，窗口会显示新的端口，例如：

```text
http://127.0.0.1:5001
```

### 3. AI 功能不可用

进入「设置」页，补齐 AI 文案和图片 Key。环境检测里的 `AI 配置 WARN` 不代表软件坏了，只代表 Key 还没填。

### 4. Amazon 预览/提交不可用

进入「设置」页，补齐 Amazon SP-API 账号。环境检测里的 `Amazon 账号 WARN` 不代表软件坏了，只代表账号还没填完整。

### 5. macOS 提示无法验证开发者

这是 macOS Gatekeeper 安全提示。处理方式：

1. 右键 `启动亚马逊2.8.command`
2. 选择「打开」
3. 在弹窗中再次点「打开」

如仍被拦截，售后人员可在终端进入解压目录执行：

```bash
xattr -dr com.apple.quarantine .
```

## 内部源码部署方式

只有内部技术/售后需要从源码部署时，才使用下面方式：

```bash
python3 tools/bootstrap.py
```

脚本会自动：

- 创建 `.venv`
- 优先使用 `vendor/wheelhouse/` 离线依赖包安装
- 没有离线依赖包时从 PyPI 在线安装
- 复制 `.env.example` 为 `.env`
- 创建运行目录
- 生成 `启动开发版.command` / `启动开发版.bat`
- 执行环境检测

## 离线依赖包

GitHub Actions 会额外生成：

- `AmazonListingTool-dependencies-darwin-arm64.zip`
- `AmazonListingTool-dependencies-windows-amd64.zip`

这些包用于「源码部署」的离线依赖安装，不是给客户直接运行的软件。客户只需要 `AmazonListingTool-*.zip`。

## 发版流程

推荐流程：

```bash
git tag v2.8.x
git push origin v2.8.x
```

推送 `v*` 标签后，GitHub Actions 会自动构建并上传：

- macOS 可执行发行包
- Windows 可执行发行包
- macOS 离线依赖包
- Windows 离线依赖包

交付给客户时，只发送对应系统的 `AmazonListingTool-*.zip`。
