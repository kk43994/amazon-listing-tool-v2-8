# 亚马逊 2.8 客户交付与新电脑部署说明

这份说明面向交付/售后人员。客户不需要理解 Python、pip、依赖、虚拟环境，客户只拿到一个压缩包，解压后双击启动脚本即可使用。

## 推荐交付方式

1. 在 GitHub Release 下载客户电脑对应的压缩包：
   - macOS Apple Silicon：`AmazonListingTool-darwin-arm64.zip`
   - Windows 64 位：`AmazonListingTool-windows-amd64.zip`
2. 把 zip 发给客户，让客户先解压，再双击：
   - macOS：`启动亚马逊2.8.command`
   - Windows：`启动亚马逊2.8.bat`
   - 如果客户电脑中文文件名显示乱码，改用 `Start-Amazon-2.8.command` / `Start-Amazon-2.8.bat`
   - 解压后根目录有 `客户先看这里.txt`，普通客户只看这份即可。
3. 程序会自动：
   - 检测运行环境
   - 创建 `.env`、`accounts.json`、`input/`、`output/`、`logs/`
   - 启动本地 Web 服务
   - 打开浏览器访问工作台
4. 第一次打开工作台会出现「首次使用引导」，让客户按顺序完成：
   - 填 AI 文案 Key / 图片 Key
   - 添加 Amazon SP-API 账号
   - 导入商品 Excel、生成示例 Excel，或生成官方模板
   - 运行一键自检
   - 工作台顶部的「只需点这个」大按钮会一直指向下一步，客户不要跳步骤
5. AI 中转配置固定按网页提示填写：
   - Base URL：`https://api.kk666.best`
   - 文字/多模态模型：`gpt-5.5`
   - 文字接口：`/v1/responses`
   - 图片模型：`gpt-image-2`
   - 图片接口：`/v1/images/generations`
   - 不会填时点击设置页的「一键恢复推荐」
   - 普通客户保持「简单模式」，只填文字 Key、图片 Key；不要进入高级模式改 endpoint/model
6. 设置页已按客户操作拆成四个标签：
   - `AI 接口`：只填 Key，一键恢复 `https://api.kk666.best`
   - `Amazon 账号`：添加 Seller ID、LWA Client ID、Secret、Refresh Token
   - `环境自检`：运行自检、复制支持信息、导出支持包
   - `高级设置`：输出目录、媒体存储等售后配置项
7. Amazon 账号点「测试」后会分项显示 LWA Token、站点映射、Listings API、权限是否通过；失败时让客户截图该弹窗。

## 客户电脑不需要安装的东西

客户电脑不需要安装以下内容：

- Python
- pip
- `requirements.txt`
- 虚拟环境 `.venv`
- PyInstaller
- 任何命令行依赖

这些内容已经被打包进 `AmazonListingTool` 可执行程序里。

## 环境检测

发行包内置环境检测入口：

- macOS：双击 `环境检测.command`
- Windows：双击 `环境检测.bat`

如果客户完全不会排查，优先使用 Doctor：

- macOS：双击 `一键检测修复.command`
- Windows：双击 `一键检测修复.bat`

Doctor 会自动补齐 `.env`、`accounts.json`、`input/output/logs/config`，并把报告保存到 `logs/doctor-report.txt` 和 `logs/doctor-report.json`。让客户把窗口截图和 `logs/doctor-report.txt` 一起发给售后即可。

如果需要进一步排查，让客户导出支持包：

- macOS：双击 `导出支持包.command`，或英文备用 `Support-Bundle.command`
- Windows：双击 `导出支持包.bat`，或英文备用 `Support-Bundle.bat`
- Web 设置页：点击「导出支持包」

支持包会保存为 `logs/support-bundle-时间.zip`，会自动脱敏 `.env` 和 `accounts.json`，可以直接让客户发给售后。

Excel 数据安全：

- 系统每次写回 Excel 前都会先备份到 `backups/`
- 系统写回时会先写临时文件并校验，再替换原文件；如果 Excel/WPS 正在占用，会提示 `EXCEL_LOCKED`，让客户关闭表格后重试
- 客户可以在设置页点「打开输出目录」和「打开最新备份」
- 发行包根目录也有 `打开输出目录` / `Open-Output`、`打开备份目录` / `Open-Backups`
- 如果客户担心真实数据，先点网页里的「生成示例 Excel」测试流程

检测项包括：

- 当前操作系统和架构
- 是否为已打包发行版
- 发行包完整性和内置 Python 依赖
- 是否在临时目录、云同步目录、网络共享目录运行
- `input/output/logs/config` 目录是否可写
- `.env` 是否存在
- `accounts.json` 是否存在、默认账号是否填完整
- AI Key 是否填完整
- AI Base URL 是否仍是旧域名或示例域名
- 默认浏览器是否可用
- Web 端口是否可用
- Doctor 模式额外按 DNS / TCP / TLS / HTTP 分层检查 `api.kk666.best`、Amazon LWA、Amazon SP-API 北美/欧洲/远东连通性
- `dependency-inventory.json` 中记录的内置依赖是否可导入

检测结果只有 `ERROR` 才会阻止启动；`WARN` 表示可以打开软件，但某些业务功能可能需要配置后才能使用。

## 端口防呆

默认端口是 `5000`。如果客户电脑上已经有其他软件占用了 `5000`，发行版会自动尝试 `5001`、`5002` 等后续端口，并在控制台输出实际访问地址。

## 常见问题处理

### 1. 双击没反应或打不开

先双击 `一键检测修复.command` / `一键检测修复.bat`，让客户把窗口截图和 `logs/doctor-report.txt` 发给技术支持。

如果启动窗口已经打开但浏览器没弹出来，让客户复制窗口里的 `http://127.0.0.1:端口` 到浏览器打开。

### 2. 浏览器没有自动打开

查看启动窗口里的地址，通常是：

```text
http://127.0.0.1:5000
```

如果端口被占用，窗口会显示新的端口，例如：

```text
http://127.0.0.1:5001
```

### 3. AI 功能不可用

进入「设置」页，保持「简单模式」，先点「一键恢复推荐」，确认 Base URL 是 `https://api.kk666.best`，再补齐 AI 文案和图片 Key。环境检测里的 `AI Key WARN` 不代表软件坏了，只代表 Key 还没填。

### 4. Amazon 预览/提交不可用

进入「设置」页，补齐 Amazon SP-API 账号。环境检测里的 `Amazon 账号 WARN` 不代表软件坏了，只代表账号还没填完整。

客户不知道每项填什么时，看设置页的「配置网址导航」和「每个输入框填什么」：

- Seller Central：查看 Seller ID / Merchant Token
- Developer Console：查看 LWA Client ID / Client Secret
- SP-API 文档：确认授权和 Refresh Token 获取流程

### 5. macOS 提示无法验证开发者

这是 macOS Gatekeeper 安全提示。处理方式：

1. 右键 `启动亚马逊2.8.command`
2. 选择「打开」
3. 在弹窗中再次点「打开」

如仍被拦截，售后人员可在终端进入解压目录执行：

```bash
xattr -dr com.apple.quarantine .
```

## 发行文件核对

构建后的客户包根目录至少应包含：

- `启动亚马逊2.8` / `Start-Amazon-2.8`
- `一键检测修复` / `Doctor`
- `环境检测` / `Env-Check`

- `打开输出目录` / `Open-Output`
- `打开备份目录` / `Open-Backups`
- `客户先看这里.txt` / `Read-Me-First.txt`
- `.env.example`
- `VERSION`
- `release-manifest.json`
- `dependency-inventory.json`
- `亚马逊商品采集模板_v1.0.xlsx`
- `docs/客户部署说明.md`
- `docs/RELEASE_CHECKLIST.md`

本地构建后可运行：

```bash
python tools/verify_release.py release/AmazonListingTool-<platform>
```

完整发版流程按 `docs/RELEASE_CHECKLIST.md` 逐项确认，尤其要在干净 Windows/macOS 机器上验证双击入口、Doctor、支持包和示例 Excel。

## 内部源码部署方式

只有内部技术/售后需要从源码部署时，才使用下面方式：

```bash
python3 tools/bootstrap.py
```

脚本会自动：

- 创建 `.venv`
- 优先使用 `vendor/wheelhouse/` 离线依赖包安装
- 没有离线依赖包时从 PyPI 在线安装
- 复制 `.env.example` 为 `.env`
- 创建运行目录
- 生成 `启动开发版.command` / `启动开发版.bat`
- 执行环境检测

## 离线依赖包

GitHub Actions 会额外生成：

- `AmazonListingTool-dependencies-darwin-arm64.zip`
- `AmazonListingTool-dependencies-windows-amd64.zip`

这些包用于「源码部署」的离线依赖安装，不是给客户直接运行的软件。客户只需要 `AmazonListingTool-*.zip`。

## 发版流程

推荐流程：

```bash
git tag v0.4.x
git push origin v0.4.x
```

推送 `v*` 标签后，GitHub Actions 会自动构建并上传：

- macOS 可执行发行包
- Windows 可执行发行包
- macOS 离线依赖包
- Windows 离线依赖包

交付给客户时，只发送对应系统的 `AmazonListingTool-*.zip`。
